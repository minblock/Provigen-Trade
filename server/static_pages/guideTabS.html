Information

Due to the higher security needs of a staking server (compared to a masternode), this script is FOR ADVANCED USERS ONLY. Please read the entire guide before you proceed. This script will try to create a safe environment for remote staking by hardening the remote server you stake on. It will also help you with choosing a good user password. VPS setup

>The VPS you plan to install your masternode on needs to have at least [X] GB of RAM and [X]GB of free disk space. We do not recommend using servers who do not meet those criteria, and your masternode will not be stable.

We recommend you to use VULTR for hosting, sign up a account at VULTR Here.

Select: Vultr Cloud Compute (VC2) -> Any location -> 64 Bit OS -> Ubuntu 16.04 -> Select correct server size -> make sure to remove any additional features -> Enter any name -> Press Deploy.

Download Bitvise SSH client -> login -> host enter: VPS IP and port 22 -> login -> copy your VPS password from VULTR. Make sure you are using root by using: sudo -i

Find the Staking script down below and press "Get script". Rightclick to paste it into the console and hit enter.

Follow the instruction in the console to proceed the installation. Paste infomation by using right click instead of Ctrl V.

Commands

During the installation, the staking script will change the order of the SSH host keys for security reasons. As a result, the next time you log into your VPS or Raspberry Pi, you will get the following warning:

systemctl start (coinname)d - Starts the wallet.
systemctl stop (coinname)d - Stops the wallet.
coinname-decrypt password - Unlock your wallet for staking .
coinname-cli getstakingstatus - Find out if staking is working.
coinname-cli getbalance - See your current balance.
coinname-cli getinfo - Find out information about your wallet.
